## General Optimization Techniques Applied

### 1. Layer Caching Strategy
- Manifests (package.json, pyproject.toml, uv.lock) copied **before** source code
- Ensures dependency layers remain cached across source-only commits
- **Impact:** 3–5 minutes saved per rebuild on source-only changes

### 2. Cache Pruning
- `npm cache clean --force` after installs
- `rm -rf /var/lib/apt/lists/*` and `/var/cache/apt/*` after APT
- `rm -rf __pycache__` and `.pytest_cache` after Python builds
- **Impact:** 50–150MB per layer

### 3. Minimal APT Footprint
- `--no-install-recommends` flag (already in place)
- Removed non-essential packages (ffmpeg)
- **Impact:** 80–100MB for optional tools

### 4. Multi-Stage Build Discipline
- Build stage includes compile tools (gcc, g++, cmake)
- Runtime stage inherits only compiled artifacts
- **Impact:** 200–300MB reduction in final image

## Deployment Recommendations

1. **Use BuildKit for faster builds:**
   ```bash
   DOCKER_BUILDKIT=1 docker build -t hermes-agent:optimized .
   ```

2. **Enable layer caching across CI/CD:**
   ```bash
   docker build --cache-from ghcr.io/your-org/hermes-agent:latest -t hermes-agent:new .
   ```

3. **Registry push strategy:**
   - Push with `latest` tag for production
   - Use semver tags for versioning
   - Example: `ghcr.io/solpi/hermes-agent:1.0.0`, `:latest`
